## Part 1 · 功能总结（Agent 能做什么）

括号里是背后用到的工具 / 技术。底层共 **7 个工具（function calling）** + **ReAct 智能体循环**。

### 一、输入与感知（多模态）
1. 文字记账
2. 语音记账（浏览器 Web Speech 转文字，需 HTTPS）
3. 截图记账（独立 OCR 服务转文字后入账）

### 二、记账核心（自然语言 → 结构化）
4. 一句话记账：解析金额/分类/账户/时间（`add_record`）
5. 一句多笔自动拆分（一次回复多个 `tool_calls`）
6. 默认值补全（账户=最常用、时间=今天、分类按内容推断）
7. 相对时间换算（“昨天/上周五”→ ISO 日期）
8. 模糊金额估算 + `pending` 待确认标记
9. 去重判断（先 `query_records` 再决定）
10. 收入/支出识别

### 三、多轮对话
11. 信息不全主动追问（澄清）
12. 多轮补充后入账

### 四、对话式账本管理（CRUD）
13. 对话式改账（先查后改，`update_record`）
14. 对话式删账 + 轻确认（人在回路，`delete_record`）
15. 账本页手动编辑/删除/按月查看

### 五、查询与分析
16. 自然语言查询（“外卖花了多少”，`query_records`）
17. 月度汇总问答（总额/分类/环比，`get_summary`）

### 六、预算与主动提醒（主动性）
18. 设预算（对话 `set_budget` 或分析页手动设）
19. 超支/接近预算 主动提醒（`budget_alert`）
20. 预算进度条（绿/黄/红）

### 七、进阶 Agent 能力
21. 目标导向省钱建议（Planning）——“怎么省钱”→ 自主分析 + 给可执行方案
22. 反思校验（Reflection）——记到远超平时的笔 → 提示核对（`anomaly_note`）
23. 个性化长期记忆（Memory）——“记住星巴克算餐饮”→ 越用越准（`remember` + `prefs` 注入）
24. AI 解读 / 异常洞察——模型生成洞察 + “某类比近 3 月均值多 N%”

### 八、可视化仪表盘（分析页）
25. 分类占比饼图 · 账户占比环形图 · 近 6 月收支对比柱 · 每日支出热力图 · 本月 Top 支出榜 · 总额/环比卡片 · 已记住偏好卡

### 底层 Agent 技术
- **7 个工具**：`add_record` / `query_records` / `get_summary` / `update_record` / `delete_record` / `set_budget` / `remember`
- **ReAct 循环**（`run_agent`：理解→决策→调工具→回灌→再循环）
- 上下文注入（今天 + 最常用账户 + 已学偏好）、结构化护栏（`normalize_fields`）、prompt 工程、dirty 联动刷新
- 质量：确定性 72 项 + 真实模型实测 22 项 = **94 项测试全绿**

## Part 2 · 演示脚本（汇报现场照着走）

### 演示前准备
- 用 **Chrome / Edge** 打开 <https://pennypal.orlando.ink>，先**强刷**一次。
- 准备一张**支付截图**（微信/支付宝账单，或任意含“商家 + 金额”文字的图）。
- 想让仪表盘一上来就满，先用 `scripts/seed_demo.py` 铺一个月样例数据（可选）。

### 第一幕 · 一句话记账（核心价值）
| 输入 | 展示的能力 |
|---|---|
| `中午吃饭35，微信付的` | 自然语言理解 + 默认值补全 |
| `早餐12 打车18 超市买菜60` | **一句多笔自动拆 3 笔** |
| 点 🎤 说「打车20」 | 语音多模态 |
| 点 📷 传支付截图 | 截图 OCR 多模态 |

### 第二幕 · 它很聪明
| 输入 | 展示 |
|---|---|
| `35` →（它反问）→ 回 `午饭` | **追问澄清 / 多轮** |
| `昨天买衣服好像两百多` | 相对时间 + 模糊估算（pending）+ 细分类「服饰」 |
| `买个电脑摄像头350 支付宝` | 细分类「数码」 |

### 第三幕 · 对话式管理（CRUD）
| 输入 | 展示 |
|---|---|
| `把刚才那笔改成300` | **对话式改账**（先查后改） |
| `删掉刚才那笔` →（它问“确定吗”）→ `确定` | **删除轻确认 / 人在回路** |

### 第四幕 · 主动 & 会学（亮点幕）
1. **预算 + 超支提醒**：`把餐饮预算设成30` → 再记 `午饭35` → 一行 ⚠️ 超支提醒。
2. **反思校验**：先连记三次 `午饭30`，再记 `聚餐花了300` → 它提示“这笔比你平时高，记错了告诉我”。
3. **记忆 / 越用越准**：`记住星巴克算餐饮` → 再记 `星巴克38` → 自动归餐饮；右侧“🧠 已记住的偏好”卡出现该条。
4. **目标导向规划**：`这个月怎么省钱？` → 它自主分析占比/环比/预算，给出**具体省钱建议**。
5. **查询**：`这个月外卖花了多少？` → 自然语言作答（不新增记录）。

### 第五幕 · 仪表盘（顺势收尾）
切到右侧：**边记边变**的 分类/账户占比、收支对比、每日热力图、Top 支出、预算进度条、记忆偏好；点 **「✨ AI 解读」** 让模型现场生成一段洞察。

### 现场技巧
- 模型有 **2~5 秒延迟**，从容讲解；偶尔没触发就**换句话再说一遍**（LLM 非确定性，正常）。
- 反复强调 **“右侧实时联动”**——左边记一笔，右边账本和图立刻更新，这是 demo 的记忆点。
- 反思要先垫 ≥3 笔同类小额；超支要先设低预算——本脚本已按顺序排好。
- 串场金句：**“传统记账记一笔要点 5~6 下，这里只要说一句话，剩下交给 Agent。”**
